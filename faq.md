# FAQ

## Что такое Mnemos простыми словами

Это система памяти для AI. Она помогает сохранять важные знания и
возвращать их AI в нужный момент.

## Это чат-бот

Нет. Это не сам чат, а слой памяти, который можно подключить к чатам,
агентам и приложениям.

## Чем это лучше обычных заметок

Заметки полезны людям, а Mnemos делает знания пригодными ещё и для AI:
их можно искать по смыслу, структурировать и использовать программно.

## Нужен ли для этого технический опыт

Для первого запуска сейчас всё ещё нужен Terminal и Docker Desktop.
После запуска уже доступен веб-интерфейс, но полностью нетехнический
путь использования ещё развивается.

## Зачем на сайте был Python и make venv

Для пользовательской установки через Docker Compose это не нужно.

`make venv` относится к сценарию разработки: локальный CLI, тесты,
pre-commit и работа с кодом вне контейнеров.

## Можно ли хранить данные у себя

Да. Проект рассчитан на локальный запуск и контроль над собственной
инфраструктурой.

## Можно ли проверять новые знания перед добавлением

Да. Для этого есть candidates и review workflow.

## Нужно ли устанавливать PostgreSQL отдельно

Нет. В локальном режиме PostgreSQL поднимается внутри Docker.

## Нужно ли устанавливать Qdrant отдельно

Нет. Он тоже запускается через Docker Compose.

## Можно ли установить это не только на macOS

Да. Если у вас есть Docker Compose, Mnemos можно запускать на macOS,
Linux и Windows.

## Что открывать после запуска

Основной пользовательский адрес:

- `http://localhost:8000/`

Технические проверки:

- `http://localhost:8000/health/live`
- `http://localhost:8000/health/ready`

## Можно ли всё делать только мышкой

Пока нет. Для первого запуска и части административных действий всё ещё
нужен Terminal.

## Что делать, если Docker Desktop не запущен

1. Откройте Docker Desktop.
1. Подождите, пока он полностью стартует.
1. Повторите команду запуска.

## Что делать, если не открывается health ready

1. Подождите немного после первого запуска.
1. Выполните `make logs`.
1. Проверьте, что все контейнеры действительно запущены.

## Как подключить Mnemos к агенту

Используйте MCP endpoint:

```text
http://localhost:9000/mcp
```

Или установите skill из репозитория:

```sh
npx skills add https://github.com/pastukhov/mnemos --skill mnemos-memory
```
