# Mnemos

## Память для ваших AI-помощников

Mnemos помогает сохранять важные факты, заметки и выводы так, чтобы AI
опирался не на догадки, а на накопленные знания. Это слой памяти между
вашими данными и AI-инструментами.

- [Что это такое](/mnemos/about/)
- [Руководство пользователя](/mnemos/guide/)
- [Установка через Docker](/mnemos/install/)

> Open source. FastAPI + PostgreSQL + Qdrant + MCP.

---

## Что решает Mnemos

Большинство AI-инструментов хорошо отвечают в рамках одного диалога, но
быстро теряют контекст. Важные детали остаются в заметках, чатах,
документах и голове пользователя.

Mnemos нужен, чтобы:

- AI не забывал важные факты и предпочтения
- знания не расползались по разным источникам
- можно было искать нужный контекст по смыслу
- новые знания добавлялись через review, а не хаотично
- длинные интервью можно было сохранять как notes без хрупких лимитов
- похожие факты можно было проверить через shortlist до записи
- память можно было подключить к агенту, а не только читать вручную

---

## Как это работает

1. Вы загружаете заметки, анкеты, тексты или ручные записи.
1. Mnemos сохраняет длинные notes как сырой материал.
1. Система может собрать shortlist candidate facts без записи.
1. Review sessions помогают проверить кандидатов пакетом.
1. AI получает нужный контекст через API или MCP.

[Подробнее о том, как устроен Mnemos](/mnemos/about/#как-это-работает)

## Где это полезно

### Личный AI-ассистент

Чтобы помощник помнил ваши цели, привычки, стиль работы и предпочтения.

### Командная база знаний

Чтобы AI учитывал процессы, термины и накопленный опыт команды.

### AI внутри продукта

Чтобы ответы были персонализированными и учитывали уже известный
контекст пользователя.

### AI-агенты и помощники

Чтобы агент мог не только отвечать, но и искать память, сохранять новые
заметки и предлагать факты через governance.

---

## Почему это лучше обычного чата

- память отделена от одной сессии
- данные лежат в вашей инфраструктуре
- есть поиск по смыслу
- есть review для новых знаний
- есть review sessions для интервью и пакетной проверки
- есть upsert-режим для обновления устойчивого профиля
- есть открытая архитектура и open source код
- есть MCP server для подключения к агентам

---

## Как подключить к агенту

Mnemos можно использовать не только как локальный интерфейс, но и как
память для агентских систем.

Основной путь подключения:

- запустить Mnemos через Docker Compose
- использовать MCP endpoint `http://localhost:9000/mcp`
- подключить этот endpoint в агенте или MCP-клиенте

Если вы работаете в Codex CLI, в репозитории уже есть installable skill
`mnemos-memory`, который помогает агенту правильно использовать память.

- [Подробнее о подключении к агенту](/mnemos/guide/#как-подключить-mnemos-к-агенту)
