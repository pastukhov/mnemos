# Mnemos: MVP для не-технического пользователя

## 1. Что такое Mnemos

Mnemos — это личная система памяти.

Она помогает:

- сохранять заметки и наблюдения
- находить их через поиск
- автоматически извлекать факты из текста
- формировать выводы из накопленных данных

Проще говоря, это **личная база знаний с AI-помощником**, который
помогает превращать заметки в структурированные знания.

---

## 2. Основные сценарии использования

### 1. Личная база наблюдений

Пользователь сохраняет мысли, решения, наблюдения и факты.

Пример:

Я предпочитаю YAML для конфигураций

---

### 2. Поиск по прошлому опыту

Через некоторое время пользователь может найти старые записи:

что я говорил про конфиги?

---

### 3. Автоматическое извлечение фактов

Система анализирует заметки и выделяет факты.

Пример:

raw: "Я предпочитаю YAML для конфигураций"
↓
fact: "Artem prefers YAML configs"

---

### 4. Review новых гипотез

Система может предложить новые выводы:

Candidate reflection:
"Artem prefers declarative configuration tools"

Пользователь решает:

- принять
- отклонить

---

## 3. Путь пользователя (User Journey)

## Первый запуск

1. пользователь открывает приложение
1. видит краткое объяснение системы
1. проходит короткий onboarding

---

## Онбординг (Wizard)

Wizard предлагает:

- добавить первую заметку
- загрузить файл
- попробовать поиск

---

## Добавление записи

Пользователь вводит текст:

Сегодня настроил Prometheus, YAML конфиги удобнее JSON.

Система сохраняет запись.

---

## Поиск

Пользователь вводит:

конфиги

Система показывает связанные записи.

---

## Review кандидатов

Система предлагает извлечённые факты.

Пользователь:

- подтверждает
- отклоняет

---

## 4. Основные функции MVP

## 4.1 Веб-интерфейс

Минимальное приложение:

/search
/add
/import
/review
/settings

---

## 4.2 Добавление записи

Пользователь может:

- написать текст
- сохранить заметку

Пример:

Я предпочитаю YAML для конфигураций

---

## 4.3 Поиск

Поиск должен работать по смыслу.

Пример:

запрос: конфиги

результат:

Artem prefers YAML configs

---

## 4.4 Импорт данных

Пользователь может загрузить файлы.

Поддерживаемые форматы:

- TXT
- Markdown
- CSV
- ChatGPT export

Интерфейс:

[ Drag & Drop файл ]

Система автоматически:

- разбивает текст
- извлекает факты
- добавляет их в память

---

## 4.5 Review кандидатов

Система показывает извлечённые кандидаты.

Candidate Fact

Artem prefers YAML configs

confidence: 0.93
source: chat transcript

[ Accept ] [ Reject ]

---

## 6. Модель данных (упрощённая)

Для пользователя система работает с четырьмя типами данных.

| Тип | Описание |
|----|----|
| Заметка | исходный текст |
| Факт | подтверждённое знание |
| Кандидат | предположение системы |
| Вывод | обобщённое знание |

---

## 7. Термины системы

| Внутренний термин | Пользовательский термин |
|---|---|
| raw | заметка |
| fact | факт |
| candidate | предположение |
| reflection | вывод |

---

## 8. Минимальный MVP

Чтобы инструментом мог пользоваться не-айтишник, достаточно:

### Интерфейс

- веб UI
- поиск
- добавление заметок
- review кандидатов

### Импорт

- drag & drop
- txt / md / csv

### Онбординг

- wizard
- объяснение системы

---

## 9. Чего НЕ нужно в MVP

- сложные настройки
- сложная модель данных
- управление pipeline
- инфраструктурные параметры

Это должно быть скрыто.

---

## 10. Архитектура (для справки)

Внутри система состоит из:

- API
- PostgreSQL
- Vector database
- LLM extraction
- pipelines обработки данных

Пользователь этого не видит.

---

## 11. Критерии готовности MVP

Инструмент готов, если пользователь может:

1. открыть приложение
1. понять что делает Mnemos
1. добавить запись
1. найти её через поиск
1. загрузить файл
1. принять предложенный системой факт

Без чтения документации.
