# Установка Mnemos на macOS с Docker Desktop

Это пошаговая инструкция для человека, у которого есть MacBook и
Docker Desktop, но нет привычки работать с серверными инструментами.

---

## Что вы получите в конце

После выполнения этой инструкции у вас будет локально запущен Mnemos:

- API на `http://localhost:8000`
- MCP endpoint на `http://localhost:9000/mcp`

---

## Перед началом

Убедитесь, что у вас есть:

- MacBook с установленной macOS
- установленный Docker Desktop
- доступ к приложению Terminal
- скачанная папка проекта `mnemos`

Если Docker Desktop ещё не установлен, сначала установите его с
официального сайта Docker.

---

## Шаг 1. Запустите Docker Desktop

1. Откройте `Applications`.
1. Запустите `Docker`.
1. Дождитесь, пока Docker Desktop покажет, что он готов к работе.

> **Важно**: пока Docker Desktop полностью не запущен, Mnemos не
> стартует.

---

## Шаг 2. Откройте Terminal

1. Нажмите `Command + Space`.
1. Введите `Terminal`.
1. Нажмите `Enter`.

Откроется окно, куда можно вставлять команды.

---

## Шаг 3. Перейдите в папку проекта

Если проект уже скачан, перейдите в его папку:

```sh
cd /путь/к/mnemos
```

Если вы не знаете точный путь, можно ввести `cd`, поставить пробел и
перетащить папку `mnemos` в окно Terminal.

Пример:

```sh
cd /Users/yourname/Downloads/mnemos
```

---

## Шаг 4. Подготовьте файл настроек

Скопируйте пример настроек:

```sh
cp .env.example .env
```

Это создаст рабочий файл `.env`, который Mnemos будет использовать при
запуске.

---

## Шаг 5. Подготовьте локальное окружение Python

Выполните:

```sh
make venv
```

Что делает эта команда:

- создаёт локальное окружение Python
- устанавливает зависимости проекта

Если команда выполняется несколько минут, это нормально при первом
запуске.

---

## Шаг 6. Установите локальные git hooks

Это необязательный шаг для обычного пользователя, но полезный, если вы
планируете что-то менять в проекте:

```sh
make install-hooks
```

---

## Шаг 7. Запустите Mnemos

Выполните:

```sh
docker compose up -d --build
```

Что произойдёт:

- Docker соберёт контейнеры проекта
- запустятся PostgreSQL, Qdrant и сам Mnemos
- сервис станет доступен локально на вашем MacBook

Первый запуск обычно дольше следующих.

---

## Шаг 8. Проверьте, что сервис работает

Откройте в браузере:

- `http://localhost:8000/health/live`
- `http://localhost:8000/health/ready`

Ожидаемый результат:

- первая страница показывает, что сервис жив
- вторая страница показывает, что сервис готов к работе

Можно также запустить автоматическую проверку:

```sh
make smoke
```

---

## Шаг 9. Что делать дальше

После запуска вы можете:

- добавить записи через API
- загрузить опросник и заметки из файлов
- выполнить поиск по памяти
- включить extraction и reflection pipeline

Для общего понимания продукта откройте
[`docs/user-guide-ru.md`](/home/artem/repos/mnemos/docs/user-guide-ru.md).

---

## Если вам нужны facts и reflections

Для этих функций часто нужен внешний LLM provider.

Вам понадобится заполнить в `.env` такие переменные:

- `EMBEDDING_BASE_URL`
- `EMBEDDING_API_KEY`
- `EMBEDDING_MODEL`
- `FACT_LLM_BASE_URL`
- `FACT_LLM_API_KEY`
- `FACT_LLM_MODEL`
- `REFLECTION_LLM_BASE_URL`
- `REFLECTION_LLM_API_KEY`
- `REFLECTION_LLM_MODEL`

Если вы хотите использовать локальный mock-режим:

```sh
cp .env.local-mock.example .env
docker compose -f docker-compose.yml \
  -f docker-compose.local-mock.yml \
  up -d --build
```

## Troubleshooting

## Docker Desktop не запущен

**Признаки**:

- команда `docker compose up -d --build` падает сразу
- Docker пишет, что daemon недоступен

**Что сделать**:

1. Откройте Docker Desktop.
1. Подождите, пока он полностью запустится.
1. Повторите команду.

## Порт уже занят

**Признаки**:

- Mnemos не стартует
- в логах есть сообщения о конфликте портов

**Что сделать**:

1. Закройте другие локальные сервисы, которые используют порт `8000`
   или `9000`.
1. Перезапустите проект.

## Не открывается `health/ready`

**Признаки**:

- `health/live` работает, а `health/ready` нет
- readiness показывает, что не готов PostgreSQL или Qdrant

**Что сделать**:

1. Подождите немного после первого запуска.
1. Выполните `make logs`.
1. Проверьте, что все контейнеры запущены в Docker Desktop.

## Не хватает знаний для работы через Terminal

**Короткий ответ**: это нормальная проблема для текущей версии.

Сегодня Mnemos можно установить на MacBook, но повседневный сценарий
для не-IT пользователя ещё требует дополнительного интерфейса. Что
именно нужно добавить, описано в
[`docs/non-technical-user-mvp-ru.md`](/home/artem/repos/mnemos/docs/non-technical-user-mvp-ru.md).

## Частые вопросы

**Нужно ли устанавливать PostgreSQL отдельно?**

Нет. В локальном режиме PostgreSQL поднимается внутри Docker.

**Нужно ли устанавливать Qdrant отдельно?**

Нет. Он тоже запускается через Docker Compose.

**Что открывать после запуска?**

С технической стороны основной адрес: `http://localhost:8000`.
Но для обычного пользователя сейчас ещё не хватает отдельного
человеческого интерфейса.

**Можно ли всё делать только мышкой?**

Пока нет. Для первого запуска и большей части действий нужен Terminal.
