# Wiki

## Что такое LLM Wiki

LLM Wiki - это слой документации, который Mnemos генерирует из уже
извлечённых фактов и reflections. Он нужен, чтобы память была удобна не
только для поиска и агентов, но и для чтения человеком.

Вместо набора разрозненных записей получается связная библиотека
страниц, которая может расти вместе с памятью.

## Как это работает

1. Вы добавляете raw-материал: заметки, ответы, тексты или другие
   исходные записи.
1. Mnemos извлекает facts и собирает reflections.
1. Wiki-страницы доступны через web UI и API; схема определяет, какие
   страницы отображать и как их обновлять.
1. LLM синтезирует markdown на основе facts, reflections и, при наличии,
   уже существующего содержимого страницы.
1. Результат записывается как обычные `.md`-файлы.

Такой подход помогает отделить источник знаний от их читабельной
представленной формы.

## Как открыть

Откройте web UI или используйте API:

- `GET /api/wiki/pages` - список страниц
- `GET /api/wiki/pages/{name}` - содержимое страницы
- `POST /api/wiki/pages/{name}/regenerate` - обновить страницу

## Как устроена schema

Файл `wiki_schema.yaml` определяет, какие страницы нужно генерировать и
по каким фильтрам собирать материал.

Обычно schema задаёт:

- имя страницы
- заголовок страницы
- описание и область охвата
- domain-фильтры
- kinds и themes, которые подходят для этой страницы

По умолчанию schema помогает создать страницы вроде карьеры, ценностей,
рабочего стиля, целей и ключевых решений.

## Пример результата

```markdown
---
title: Карьера и навыки
generated_at: 2026-04-10T12:00:00Z
source_fingerprint: abc123
facts_count: 12
reflections_count: 3
---

# Карьера и навыки

## Кратко

Эта страница собирает факты о профессиональном опыте, навыках и
предпочтениях в работе.

## Основные темы

- опыт в конкретных ролях
- сильные стороны и навыки
- рабочие предпочтения
```

## Что дальше

Wiki можно использовать как слой для чтения, навигации и последующего
расширения знаний. Она не заменяет facts и reflections, а делает их
удобнее для человека.
